Troubleshooting Common Issues

Developing robotic applications is complex, and encountering issues is a natural part of the process. This appendix provides a guide to troubleshooting common problems you might face when working with ROS 2, Gazebo, NVIDIA Isaac Sim, and other tools covered in this book.

ROS 2 Errors and Fixes​

ProblemCauseSolution
ros2: command not foundROS 2 environment not sourced.source /opt/ros/humble/setup.bash (or your workspace setup.bash). Add to ~/.bashrc for permanent effect.
Nodes not discovering each other- Incorrect ROS_DOMAIN_ID.
- Firewall blocking DDS ports.
- Network issues.		- Ensure ROS_DOMAIN_ID is consistent across all devices.
- Check firewall rules (allow UDP ports 7400-7500).
- Verify network connectivity.
ImportError: No module named 'rclpy'Python environment not correctly set up.- Activate your Python virtual environment.
- Ensure rclpy (and other ROS 2 Python packages) are installed and your workspace is sourced.
Failed to load plugin 'lib...so'Missing shared library or incorrect path.- Ensure the plugin package is built and sourced.
- Check if the library (.so file) exists in your install/ directory.
- Verify plugin name and path in launch files/configurations.
High CPU/Memory Usage- Too many nodes running.
- Inefficient code.
- Large message sizes.		- Consolidate nodes where appropriate.
- Optimize code (e.g., use numpy for image processing).
- Reduce message frequency or use smaller message types if possible.
ROS 2 launch file fails to find package- Package not built.
- Package not in ROS_PACKAGE_PATH.		- Run colcon build --packages-select <package_name> and source install/setup.bash.
- Verify package.xml is correct.
Parameter not found- Parameter not declared in node.
- Incorrect parameter name/namespace.		- Ensure self.declare_parameter() is used in your node.
- Check parameter file path and namespace in launch files.

Gazebo Crashes​

ProblemCauseSolution
Gazebo crashes on startup- GPU driver issues.
- Incompatible Gazebo version.
- Corrupted installation.		- Update/reinstall NVIDIA GPU drivers.
- Ensure Gazebo Harmonic is correctly installed for Ubuntu 22.04 and ROS 2 Humble.
- Try reinstalling Gazebo.
Robot models appear distorted/invisible- Incorrect URDF/SDF syntax.
- Missing mesh files.
- Incorrect paths to models.		- Validate your URDF/SDF using check_urdf or online validators.
- Ensure mesh files (DAE, STL, OBJ) are correctly referenced and accessible.
- Check Gazebo's model path (GAZEBO_MODEL_PATH).
Simulation runs too slow- Complex world/robot model.
- High sensor update rates.
- Inefficient physics.		- Simplify models (reduce polygon count).
- Reduce sensor update rates.
- Adjust physics max_step_size or choose a different physics engine (e.g., ODE for speed).
- Ensure sufficient CPU/GPU.
Robot falls over unexpectedly- Incorrect inertial properties.
- Unstable joint limits.
- Physics engine issues.		- Double-check mass and inertia tensor values in URDF/SDF.
- Verify joint limits and stiffness.
- Experiment with different physics engines (Bullet/DART might be more stable).

Isaac Sim GPU Issues​

ProblemCauseSolution
Isaac Sim fails to launch/crashes- Outdated NVIDIA drivers.
- Insufficient GPU VRAM.
- Incorrect Omniverse/Isaac Sim installation.		- Ensure NVIDIA drivers are up-to-date.
- Close other GPU-intensive applications.
- Reinstall Isaac Sim and Omniverse Launcher, ensuring all dependencies are met.
Poor performance/low FPS- Complex scene.
- High sensor output.
- Limited GPU resources.		- Simplify 3D models and textures.
- Reduce sensor resolutions/update rates.
- Upgrade GPU or use a less demanding rendering quality setting.
CUDA Error / GPU memory issues- Running out of GPU memory.
- Incompatible CUDA version.		- Reduce scene complexity.
- Use smaller batch sizes for AI model training.
- Verify your CUDA toolkit version matches Isaac Sim requirements.

Network Problems​

ProblemCauseSolution
Devices not communicating (ROS 2)- Different ROS_DOMAIN_ID.
- Firewall.
- Network configuration.		- Set ROS_DOMAIN_ID to the same value on all devices.
- Configure firewall to allow DDS traffic (UDP ports 7400-7500).
- Ensure devices are on the same subnet, check IP addresses.
High latency/packet loss- Congested network.
- Poor Wi-Fi signal.
- Large message sizes.		- Use wired Ethernet where possible.
- Optimize Wi-Fi setup (clear channels, stronger signal).
- Reduce message frequency or use more efficient message types.
SSH/SCP connection refused- Incorrect IP/username.
- SSH server not running.
- Firewall on remote host.		- Double-check IP address and username.
- Ensure SSH server is installed and running on the target device (sudo systemctl status ssh).
- Configure firewall on target to allow SSH (port 22).

Build Failures​

ProblemCauseSolution
colcon build fails- Missing dependencies.
- Syntax errors in code.
- Incorrect package.xml/setup.py.		- Run rosdep install --from-paths src --ignore-src -r -y.
- Carefully check compiler/interpreter errors.
- Validate XML/Python syntax, ensure build_type is correct.
npm install fails- Network issues.
- Outdated Node.js/npm.
- Corrupted package-lock.json.		- Check internet connection.
- Update Node.js and npm (nvm install node && nvm use node).
- Delete node_modules and package-lock.json, then retry npm install.
Docusaurus build fails (npm run build)- Broken internal links.
- Markdown/MDX syntax errors.
- Incorrect Docusaurus config.		- Check console output for specific errors (e.g., broken links, markdown parsing issues).
- Ensure all Markdown/MDX files are correctly formatted and frontmatter is valid.
- Review docusaurus.config.js.
